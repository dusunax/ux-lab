# Notion Daily 하네스

**인수:** $ARGUMENTS

## Step 0 — 인수 파싱

`$ARGUMENTS`에서 추출:

| 패턴 | 별칭 | 동작 |
|------|------|------|
| (없음) | | 오늘 페이지 생성 또는 URL 반환 |
| `--new` | `-n` | 오늘 페이지가 이미 있어도 새 페이지 강제 생성 (`-2`, `-3` suffix) |
| `--update` | `-u` | 현재 작업 자동 생성 (헤딩 + staged diff 기반) |
| `--create` | `-c` | 최근 커밋 내용을 H2 + 불렛포인트로 오늘 페이지에 추가 |
| `--log "내용"` | `-l "내용"` | 고민 사항 로그 추가 |

별칭은 `--update` = `-u`, `--create` = `-c`, `--log` = `-l`, `--new` = `-n` 으로 동일하게 처리한다.

예시:
- `/notion-daily`
- `/notion-daily -n`
- `/notion-daily -u`
- `/notion-daily -c`
- `/notion-daily -l "Tailwind v4 마이그레이션 시 토큰 충돌 가능성 있음"`

---

## 사전 조건 — Notion MCP 연결 확인

이 명령은 **Notion MCP 도구**(`mcp__notion__*`)를 사용한다.
MCP가 연결되지 않은 경우 사용자에게 안내한다:
> "Notion MCP가 연결되어 있지 않습니다. Claude Code 설정에서 Notion MCP를 추가하고 인증해 주세요."

---

## 설정 — 부모 페이지

일지를 생성할 부모 페이지 URL 또는 ID를 아래에 기입한다.
비어있으면 워크스페이스 루트(Private)에 생성된다.

```
NOTION_DAILY_PARENT_URL=
```

---

## Step 1 — 오늘 날짜 계산

```bash
date +"%y%m%d"
```

결과 예: `260415` — 이것이 페이지 제목이다.

---

## Step 2 — 오늘 페이지 검색

`mcp__notion__notion-search` 도구를 호출한다:

```
query: <오늘 날짜 yymmdd>
filter: { property: "object", value: "page" }
```

결과에서 `title`이 오늘 날짜 패턴과 일치하는 페이지를 모두 수집한다.

- `--new` **없이** 오늘 페이지가 이미 있으면: 가장 최근 것의 URL을 기억하고 Step 3·4를 건너뛴다.
- `--new` **있거나** 오늘 페이지가 없으면: Step 3으로 진행한다.

> **중복 방지 규칙**: `--new` 여부와 무관하게 기존 페이지는 절대 덮어쓰지 않는다.
> 새 페이지가 필요하면 `-2`, `-3` 순으로 suffix를 붙여 생성한다.
> 예: `260415` 존재 → `260415-2` 생성, `260415-2`도 존재 → `260415-3` 생성

---

## Step 3 — 어제 페이지의 미완료 항목 수집

`mcp__notion__notion-search` 도구로 어제 날짜(`yymmdd`) 페이지를 검색한다.

```bash
date -v-1d +"%y%m%d"   # macOS
# 또는
date -d "yesterday" +"%y%m%d"   # Linux
```

어제 페이지가 존재하면 `mcp__notion__notion-fetch`로 내용을 가져와 **체크 해제(`- [ ]`)된 to-do 항목**만 추출한다.

미완료 항목은 뎁스에 따라 heading으로 변환하여 오늘 페이지의 **오늘의 주요 할 일** 섹션에 삽입한다:

| 체크박스 뎁스 | 변환 형식 |
|---|---|
| 1뎁스 (최상위) | `# 항목명` (H1) |
| 2뎁스 (하위) | `## 항목명` (H2) |
| 3뎁스 (하하위) | `### 항목명` (H3) |

예시 — 어제 미완료:
```
- [ ] Custom Parser
    - [ ] 작업 시작
        - [ ] 파싱방식 선택 팝업
```
→ 오늘 페이지:
```
# Custom Parser
## 작업 시작
### 파싱방식 선택 팝업
```

- 어제 페이지가 없거나 미완료 항목이 없으면 기본 플레이스홀더를 사용한다.

---

## Step 4 — 오늘 페이지 생성 (Step 2에서 미발견 시에만)

`mcp__notion__notion-create-pages` 도구를 호출한다.

- `NOTION_DAILY_PARENT_URL`이 설정되어 있으면 `parent.page_id`로 사용한다.
- 비어있으면 `parent`를 생략해 워크스페이스 루트에 생성한다.
- 제목은 Step 2의 중복 방지 규칙에 따라 결정한다 (예: `260415`, `260415-2`, …).

```
pages:
  - properties:
      title: <결정된 제목 (yymmdd 또는 yymmdd-N)>
    content: |
      # Todo
      - [ ] <어제 1뎁스 미완료 항목>      ← 모든 뎁스의 미완료 항목을 계층 그대로 포함
        - [ ] <어제 2뎁스 미완료 항목>
          - [ ] <어제 3뎁스 미완료 항목>
      - [ ] <어제 1뎁스 미완료 항목2>
        - [ ] ...

      ← 없으면 기본값:
      - [ ] 오늘의 주요 할일
      - [ ] 오늘의 주요 할일2

      ---

      # <어제 1뎁스 미완료 항목>          ← 뎁스 → 헤딩 변환 결과도 모두 포함
      ## <어제 2뎁스 미완료 항목>
      ### <어제 3뎁스 미완료 항목>
      # <어제 1뎁스 미완료 항목2>
      ## ...

      ← 없으면 기본값:
      # 오늘의 주요 할 일
      ## 부제목
      ### 세부 항목
```

응답에서 URL을 저장한다.

---

## Step 5 — 내용 추가 (인수가 있을 경우에만)

Step 2 또는 3에서 얻은 페이지 URL로 `mcp__notion__notion-update-page` 도구를 호출한다.

현재 시각은 `date +"%H:%M"` 으로 구한다.

### `--update` / `-u` 인 경우 (자동 생성)

아래 순서로 컨텍스트를 수집한 뒤 update 내용을 자동 작성한다:

**1. 오늘 페이지의 마지막 H1·H2·H3 확인**

`mcp__notion__notion-fetch`로 오늘 페이지를 가져와 마지막으로 등장하는 H1, H2, H3 헤딩을 추출한다.
이것이 현재 작업 맥락이다.

**2. 스테이지된 코드 변경점 확인**

```bash
git diff --staged
```

변경된 파일 목록과 주요 diff 내용을 파악한다.
스테이지된 변경이 없으면 `git diff HEAD`로 fallback한다.

**3. update 내용 자동 작성**

수집한 컨텍스트(현재 작업 헤딩 + 코드 변경점)를 바탕으로 간결한 불렛 포인트 요약을 작성한다.

```
page_id: <오늘 페이지 URL>
content 추가:
  ---
  - (HH:MM) <변경 항목 1>
  - <변경 항목 2>
  - ...
```

### `--create` / `-c` 인 경우

아래 순서로 최근 커밋을 읽어 오늘 페이지에 섹션을 추가한다:

**1. 최근 커밋 조회**

```bash
git log --oneline -10
```

가장 최근 커밋(또는 아직 기록하지 않은 커밋들)을 가져온다.

**2. prefix 제거 및 H2 제목 추출**

커밋 메시지에서 conventional commit prefix(`feat:`, `fix:`, `docs:`, `chore:`, `refactor:` 등)를 제거하고 나머지를 H2 제목으로 사용한다.

예: `feat: 파싱방식 선택 팝업 구현` → `## 파싱방식 선택 팝업 구현`

**3. 커밋 diff 요약**

```bash
git show --stat <커밋해시>
```

변경된 파일과 내용을 파악해 핵심 변경사항을 불렛포인트로 요약한다.

**4. 오늘 페이지에 추가**

```
page_id: <오늘 페이지 URL>
content 추가:
  ## <prefix 제거한 커밋 제목>
  - <변경 요약 불렛 1>
  - <변경 요약 불렛 2>
  - ...
```

---

### `--log "내용"` / `-l "내용"` 인 경우

```
page_id: <오늘 페이지 URL>
content 추가:
  ---
  ## 💭 고민 (HH:MM)
  - <사용자가 입력한 내용 — 불렛 포인트로 작성>
```

---

## Step 6 — 완료 보고

```
노션 일지 {created|updated|logged}: <오늘 날짜>
URL: <페이지 URL>
```

MCP 오류 발생 시 오류 메시지를 그대로 출력한다.
