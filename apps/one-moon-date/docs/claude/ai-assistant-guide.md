# AI 어시스턴트 사용 가이드

이 문서는 Claude, ChatGPT 등 AI 어시스턴트를 사용할 때
**효율적으로 작업하고 토큰을 절약하기 위한 실전 가이드**이다.

본 프로젝트(달력한칸 / one-moon-date)에 적용되는 실전 기준으로 작성되었다.

---

## 🎯 기본 원칙

1. **한 번에 모든 걸 요구하지 않는다**
2. **이미 작성된 문서는 반복해서 설명시키지 않는다**
3. **"전체 코드 다시 작성"을 최소화한다**
4. **수정은 항상 범위를 명확히 지정한다**
5. **단순한 작업에는 가벼운 모델을 사용한다**

---

## 1️⃣ 모델 선택 전략

### 가벼운 모델 사용 (단순 작업)

다음과 같은 작업에는 가벼운 모델을 사용하여 토큰을 절약하세요:

- ✅ 코드 포맷팅 및 스타일 수정
- ✅ 작은 버그 수정 (1-2줄 변경)
- ✅ 주석 추가/제거
- ✅ 변수명/함수명 변경
- ✅ 간단한 리팩토링 (함수 분리, 컴포넌트 추출)
- ✅ 파일 이동/이름 변경
- ✅ 간단한 타입 수정
- ✅ 작은 UI 스타일 조정

**예시:**
```
"이 함수의 변수명을 더 명확하게 바꿔줘"
"주석을 제거해줘"
"코드 포맷팅만 해줘"
```

### 강력한 모델 사용 (복잡한 작업)

다음과 같은 작업에는 더 강력한 모델을 사용하세요:

- ✅ 아키텍처 설계 및 큰 리팩토링
- ✅ 새로운 기능 추가 (복잡한 로직)
- ✅ 여러 파일에 걸친 수정
- ✅ 알고리즘 구현
- ✅ 복잡한 상태 관리 설계
- ✅ 성능 최적화 전략 수립

**예시:**
```
"전체 앱 구조를 리팩토링해줘"
"새로운 음력 계산 알고리즘을 구현해줘"
```

## 2️⃣ 문서 기반으로 대화하기

### ❌ 나쁜 예

```
이 프로젝트 다시 설명해줘
```

### ✅ 좋은 예

```
docs/claude.md 문서를 기준으로
Android 위젯 부분만 구현해줘
```

**핵심:** 이미 md에 있는 내용은 다시 설명시키지 않는다.

---

## 3️⃣ "전체 코드" 요청을 피하자

### ❌ 나쁜 예

```
전체 코드를 다시 작성해줘
```

### ✅ 좋은 예

```
Android AppWidgetProvider 코드만 작성해줘
```

또는

```
기존 코드에서 날짜 갱신 로직 부분만 수정해줘
```

**핵심:** 범위를 좁힐수록 토큰 사용량 ↓ 정확도 ↑

---

## 4️⃣ 출력 형식을 미리 제한하기

### ❌ 나쁜 예

```
설명도 해주고 코드도 다 보여줘
```

### ✅ 좋은 예

```
설명은 5줄 이내로,
코드는 필요한 파일만 보여줘
```

또는

```
설명 없이 코드만 출력해줘
```

---

## 5️⃣ 이미 결정된 사항은 다시 묻지 않기

이 프로젝트에서 이미 결정된 것들:
- Android 전용
- React Native
- Expo 미사용
- JavaScript 사용

### ❌ 나쁜 예

```
Expo로 할까?
```

### ✅ 좋은 예

```
Expo 없이 Android 네이티브 위젯을 추가해줘
```

---

## 6️⃣ 단계별로 쪼개서 요청하기

### ❌ 나쁜 예

```
위젯, 음력 계산, UI 전부 만들어줘
```

### ✅ 좋은 예

```
1단계: 음력 날짜 계산 로직만 구현해줘
```

다음 요청에서:

```
이 로직을 Android 위젯에서 쓰는 방법 알려줘
```

---

## 7️⃣ "변경 사항만" 요청하기

### ❌ 나쁜 예

```
이 코드 더 좋게 바꿔줘
```

### ✅ 좋은 예

```
이 코드에서
날짜 변경 감지 로직만 개선해줘
나머지는 그대로 유지해줘
```

---

## 8️⃣ 반복 출력 방지 문장 (강력 추천)

아래 문장은 **항상 프롬프트 마지막에 붙여도 좋다**:

```
이미 설명한 내용은 반복하지 말고,
새로 필요한 부분만 답변해라.
```

또는 더 강하게:

```
중복 설명 없이,
요청한 결과물만 출력해라.
```

---

## 9️⃣ Claude Code에서 특히 좋은 패턴

```
docs/claude.md를 기준으로 한다.
설명은 최소화하고,
Android 위젯 관련 코드만 작성해줘.

이전 답변에서 작성한 코드가 이미 있다고 가정하고,
수정이 필요한 부분만 보여줘.
```

## ✅ 추천 프롬프트 템플릿

### 템플릿 1: 단순 수정

```
[파일 경로]의 [함수/컴포넌트]에서
[구체적인 변경 사항]만 수정해줘.
설명 없이 코드만 보여줘.
```

### 템플릿 2: 새 기능 추가

```
docs/claude.md를 기준으로 한다.
[새로운 기능]을 구현해줘.
[관련 파일 경로]에 추가하면 된다.

설명은 최소화하고 코드 중심으로 작성해줘.
```

### 템플릿 3: 리팩토링

```
[대상 파일/폴더]를
[목표 구조]로 리팩토링해줘.

기존 기능은 유지하고,
코드만 보여줘.
```

### 템플릿 4: 버그 수정

```
[문제 설명]
[관련 파일 경로]

이 부분만 수정해줘.
```

## 📊 토큰 절약 체크리스트

요청 전에 확인:

- [ ] 이미 문서에 있는 내용을 반복 설명하고 있나?
- [ ] 전체 코드를 요청하는 대신 범위를 좁혔나?
- [ ] 단순한 작업인데 강력한 모델을 사용하려고 하나?
- [ ] 출력 형식을 제한했나?
- [ ] 반복 출력 방지 문장을 추가했나?

## 💡 실전 예시

### 예시 1: 변수명 변경 (가벼운 모델)

```
src/utils/lunarCalendar.js의
`calcLunar` 함수명을 `calculateLunarDate`로 변경해줘.
설명 없이 수정된 코드만 보여줘.
```

### 예시 2: 컴포넌트 분리 (가벼운 모델)

```
src/App.js에서
PickerSelect 컴포넌트를 별도 파일로 분리해줘.
src/components/PickerSelect/PickerSelect.js에 생성하면 된다.
```

### 예시 3: 큰 리팩토링 (강력한 모델)

```
docs/claude.md를 기준으로 한다.
전체 앱 구조를 기능별로 폴더링해줘.

components/, hooks/, screens/ 폴더로 분리하고,
각 파일의 역할을 명확히 해줘.
```

## 🎓 학습 포인트

1. **작은 단위로 나누기**: 큰 작업을 작은 단위로 나누면 토큰 절약 + 정확도 향상
2. **문서 활용**: 이미 작성된 문서를 참조하도록 지시
3. **모델 선택**: 작업 복잡도에 맞는 모델 선택
4. **명확한 지시**: 모호한 요청보다 구체적인 요청이 더 좋은 결과를 만든다
