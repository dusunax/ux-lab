Sprint 목표

URL 하나, 팀 고유 평가표 하나

URL 입력만으로 팀별 평가표와 질문 시트를 자동 생성하는 미니 심사위원 MVP를 완성한다.
  • 1 심사위원 프로필 설정 — 이름 입력, 주니어 / 미드 / 시니어 레벨 선택
  • 2 팀 URL 입력 — GitHub README + Notion 공개 페이지 URL 등록, 팀 목록 관리
  • 3 자동 파싱 — GitHub raw fetch (main→master), Notion HTML 스크레이핑. 실패 시 수동 보강
  • 4 AI 생성 — openrouter-proxy(model:auto) 연동, 레벨별 프롬프트 분기로 평가표 + 질문 시트 즉시 생성
  • 5 출력 / 인쇄 — 팀 결과 카드 + 질문 시트(기술 3 + 포괄 2) 렌더링, print CSS 팀당 1페이지 PDF 내보내기

단계 동작 상세 분류
1 심사위원 프로필 이름 + 레벨(주니어/미드/시니어) 설정 UX
2 팀 목록 입력 제목·설명·GitHub URL·Notion URL — n개 추가/삭제, 상태 뱃지 FE
3 자동 파싱 GitHub raw fetch (main→master 자동 시도) + Notion HTML 스크레이핑. 실패 시 수동 보강 모달 파싱
4 AI 생성 openrouter-proxy(model:auto) + 레벨별 프롬프트 분기. 평가표 + 질문 시트(기술 3 + 포괄 2) 생성 AI
5 출력 / 인쇄 팀 결과 카드(요약·스택·완성도) + 질문 시트 렌더링 + print CSS 팀당 1페이지 PDF 출력
핵심 차별점: 부트캠프 출품회 부스를 돌며 각 팀 URL 하나만으로 팀 고유 맥락을 즉시 파악 — 심사위원 레벨에 따라 질문 관점이 달라진다.

# 결정 내용
1 앱 위치: apps/mini-judge/ — Vite + React + TypeScript + Tailwind + Zod
2 심사위원 레벨 3단계: 주니어 / 미드 / 시니어 — 레벨별 프롬프트 분기
3 Notion 파싱: 공개 URL HTML 스크레이핑, 실패 시 수동 보강 textarea fallback
4 AI API: 기존 openrouter-proxy 재사용, model:"auto" 무료 모델 자동 폴백
5 GitHub Private 레포: raw fetch 실패 시 수동 텍스트 입력 fallback
6 질문 시트: 기술 3개 + 포괄 2개, 팀 고유 내용 최소 1개 포함
7 인쇄 내보내기: print CSS 기반, 팀당 1페이지
8 파싱 실패 UX: 실패 항목 명시 + 수동 보강 입력 모달

기술 스택

  • Frontend — React 18, TypeScript (strict), Vite, Tailwind CSS 3, Zod
  • AI — openrouter-proxy (NestJS) + OpenRouter 무료 모델 자동 폴백
  • 파싱 — browser fetch (GitHub raw) + HTML 스크레이핑 (Notion)
재사용: openrouter-proxy는 기존 모노레포 내 NestJS 서비스를 그대로 연동 — 별도 백엔드 신규 구축 없음
