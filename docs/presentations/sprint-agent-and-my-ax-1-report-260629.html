흐름

사용자 핵심 플로우

1
Home Feed
Agent 목록, 검색, 카테고리, 정렬 UI로 발견성을 검증한다.
2
Agent Detail
소개, 사용 방법, 예시, 업데이트 이력, 댓글을 한 화면에 모은다.
3
Run Result
입력 요약, 처리 단계, 공통 결과 카드, 후속 CTA를 Agent 공통 형태로 제공한다.
4
Ranking
개인/팀 랭킹으로 사내 공유와 선의의 경쟁 구조를 보여준다.
Sprint 1의 가장 중요한 검증 포인트는 Run 결과 화면이다. 단순 text 출력이 아니라 업무 결과물처럼 쓸 수 있는 구조화된 결과 카드로 개선했다.

아키텍처

구현 구조

Frontend Routes
  • / Home Feed
  • /agent/[id] Agent Detail
  • /agent/[id]/run Run Agent 결과 중심 화면
  • /ranking 개인/팀 랭킹
Backend Mock API
  • GET /api/agents 목록 · 검색 · 정렬
  • GET /api/agents/[id] 상세 데이터
  • POST /api/agents/[id]/run 구조화된 실행 결과
  • GET /api/rankings 랭킹 데이터
State Strategy
  • 좋아요, 써봤어요, Fork, 댓글은 Sprint 1 local state
  • 실제 저장소는 Sprint 2 Firestore 전환
Result Design
  • 입력 요약 → 처리 단계 → 공통 결과 카드
  • 복사, 내보내기, 써봤어요, 다시 실행 CTA 제공
Agent Metadata
  • platform: ChatGPT, Claude, Gen.AI 등 실행 위치
  • usageGuide: 가장 잘 동작하는 사용법
  • visibility: 전체 공개, 팀 공개, 비공개
