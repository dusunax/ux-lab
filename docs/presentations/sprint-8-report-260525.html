수용 기준

Acceptance Criteria

  • 유효한 Firebase ID Token 없는 /api/chat 요청은 401을 반환한다
  • 위조 / 만료 토큰 요청도 401을 반환한다
  • 유효 토큰 요청은 기존과 동일하게 OpenRouter 호출로 진행된다
  • FIREBASE_SERVICE_ACCOUNT가 하드코딩 없이 Vercel에서 로드된다
  • 클라이언트가 getIdToken()으로 토큰을 발급해 헤더에 첨부한다
  • Firestore model_labels에 운용 모델 6개가 seeding되어 있다
  • 인증된 사용자는 model_labels를 읽을 수 있고, 쓰기는 차단된다
  • 레거시 entry modelLabel 전수 마이그레이션 완료 — 3건 업데이트, model 없는 8건은 집계 제외
  • loadModelLabels() / modelLabelCache 제거 — 런타임 Firestore 조회 불필요
  • FEEDBACK_MIN_SAMPLE 복원 여부가 GA4 수치 기반으로 문서화된다
  • 프로덕션 전체 플로우 (일기 → AI → 피드백 → 대시보드) 오류 없이 동작한다
