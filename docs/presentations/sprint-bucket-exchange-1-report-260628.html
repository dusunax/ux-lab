기술 현황

아키텍처 — Next.js + Firestore

API Routes (서버)
  • GET /api/quests — Firestore 목록 조회 (최신순 50개)
  • POST /api/quests — 의뢰 등록, Bearer 토큰 검증
  • POST /api/applications — 지원서 제출, 중복 방지 (409)
Firestore 컬렉션
  • quests/{id} — 제목, 카테고리, 보상, 상태, 지원자 수
  • quests/{id}/applications/{id} — 닉네임, 이유, 계획, 일정
  • applicantCount — 지원 시 increment(1) 원자적 증가
클라이언트 Auth
  • signInAnonymously() — 앱 진입 시 자동 실행
  • useAuth() 훅 → getIdToken() 제공
  • API 호출 시 Authorization: Bearer {token} 첨부
페이지 렌더링 전략
  • Quest Board: Server Component → adminDb 직접 조회
  • Quest Detail: Server Component → doc(id) 단일 조회
  • Apply Form: Server(데이터) + Client(폼 인터랙션) 분리
