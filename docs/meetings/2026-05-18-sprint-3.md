## 로그 수집 환경 방안 (Chase 제안)

**제안일:** 2026-05-18  
**제안자:** Chase (SRE)

### 수집 면 (2개)

| 수집 위치 | 도구 |
|---|---|
| 클라이언트 (UX 이벤트) | Firebase Analytics |
| 서버 (신뢰성 이벤트) | Vercel Runtime Logs (structured JSON) |

Firestore 로그 컬렉션은 기각 — 쓰기 비용, 쿼리 UX 부재, 이미 GA/Vercel이 무료로 제공.

### 수집 이벤트

**클라이언트 (Firebase Analytics)**

| 이벤트 | 주요 파라미터 | 목적 |
|---|---|---|
| `auth_signin_attempt` | `method=google` | OAuth 마찰 감지 |
| `auth_signin_success` | `method`, `is_new_user` | 전환율 |
| `auth_signin_failure` | `code` | 인증 오류 추적 |
| `auth_signout` | — | 세션 길이 추정 |
| `entry_submit_attempt` | `text_length` | 퍼널 진입 |
| `entry_submit_success` | `text_length`, `emotion`, `latency_ms` | 퍼널 완료 + 성능 |
| `entry_submit_failure` | `reason` | 실패 원인 분류 |
| `entry_delete` | — | 참여도 |
| `entry_load_failure` | `code` | Firestore 읽기 오류 |
| `entry_save_failure` | `code` | Firestore 쓰기 오류 |

**서버 (Vercel Runtime Logs)**

| 이벤트 | 주요 필드 |
|---|---|
| `chat_request` | `request_id`, `model_requested`, `is_auto` |
| `chat_fallback` | `request_id`, `from_model`, `to_model`, `attempt` |
| `chat_upstream_error` | `request_id`, `model`, `status`, `latency_ms` |
| `chat_success` | `request_id`, `model_used`, `attempt_count`, `latency_ms` |
| `chat_exhausted` | `request_id`, `attempts` |

`x-request-id` 헤더로 클라이언트-서버 이벤트 연결.

### 절대 수집 금지 (개인정보)

- 일기 텍스트 (`entry.text`) — 길이(`text_length`)만 기록
- AI 공감 응답 (`entry.empathy`)
- 이메일 · 이름 · UID 원문
- OpenRouter 요청/응답 본문 — 토큰 수만 기록
- Firebase 오류 `message` — `code`만 기록

### 구현 우선순위

| 순위 | 항목 | 예상 공수 |
|---|---|---|
| P0 (이번 스프린트) | `chat_fallback/exhausted/success` (서버) | ~30줄 |
| P0 (이번 스프린트) | `entry_submit_success/failure` (클라이언트) | ~6줄 |
| P0 (이번 스프린트) | `auth_signin_success/failure` (클라이언트) | ~4줄 |
| P1 (다음 스프린트) | `x-request-id` 연결, Firestore 오류 이벤트 | — |
| P2 (DAU 50+ 이후) | `entry_delete`, `sheet_open`, 감정 분포 | — |
