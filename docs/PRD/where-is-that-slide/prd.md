# Where Is That Slide — PRD

> v0.1 · 2026-06-29 · PM Jordan

---

## 1. 개요

**Where Is That Slide**는 사내 Google Drive에 저장된 발표자료(PDF, PPT)를 자연어로 검색하고, 관련 슬라이드 페이지까지 찾아주는 AI 문서 탐색 서비스다.

> **"그 PPT 어디 있었지?"를 5초 만에 해결한다.**

정확한 파일명이나 키워드를 몰라도 의미 기반(Semantic) 검색으로 원하는 문서와 슬라이드를 즉시 찾아준다.

---

## 2. 문제 정의

### 현재 상황

사내에서는 다수의 발표자료와 교육 자료가 Google Drive에 축적되어 있다.
그러나 기존 Drive 검색은 **정확한 파일명 또는 키워드**를 알아야만 원하는 문서를 찾을 수 있다.

### 반복 발생 상황

| 상황 | 현재 해결 방식 |
|------|---------------|
| "OCR 발표자료 어디 있었지?" | 여러 폴더를 직접 탐색 |
| "지난 OKR 발표 다시 보고 싶은데…" | 작성자에게 직접 요청 |
| "신입 교육 때 봤던 Chunking 자료가 뭐였더라?" | 슬랙에서 공유 링크 검색 |
| "Document AI 소개 PPT 좀 찾아주세요." | 팀 내 전파 요청 |

### 핵심 문제

1. 파일명 의존 검색 — 기억하지 못하면 검색 불가
2. 슬라이드 내용 미검색 — 파일 내부 텍스트까지 탐색 불가
3. 담당자 의존 — 문서 소재 파악이 사람에게 집중됨

---

## 3. 목표 및 성공 지표

### 목표

자연어 질문 하나로 관련 문서와 슬라이드 페이지를 5초 내에 제시한다.

### 성공 지표 (Sprint 1 기준)

| 지표 | 목표값 |
|------|--------|
| 검색 응답 시간 | p95 ≤ 5초 |
| 검색 정확도 (top-3 relevant) | ≥ 80% (파일럿 수동 평가) |
| 슬라이드 번호 제시 정확도 | ≥ 70% |
| 인덱싱 커버리지 | 파일럿 문서 100% |

## 7. 시스템 아키텍처

### 인덱싱 파이프라인

```
Google Drive
  └─ 파일 목록 조회 (Drive API)
       └─ 파일 다운로드 (PDF / PPT)
            └─ 텍스트 추출
                 ├─ PDF: pdfplumber (페이지 단위)
                 └─ PPT: python-pptx (슬라이드 단위)
                      └─ 청킹
                           ├─ PPT: 슬라이드 1개 = 청크 1개
                           └─ PDF: 페이지 단위, 50자 미만 페이지는 인접 페이지와 병합
                                └─ OpenAI Embedding 생성
                                     └─ ChromaDB 저장
                                          (파일명, 페이지 번호, Drive URL 메타데이터 포함)
```

### 검색 파이프라인

```
사용자 질문 (자연어)
  └─ OpenAI Embedding 생성
       └─ ChromaDB Semantic Search (top-k)
            └─ OpenAI Responses API (요약 생성)
                 └─ 응답: [파일명, 추천 페이지, 요약, Drive 링크]
```
