# AI 응답 기준선 샘플 (Baseline Samples)

> 작성일: 2026-05-20
> 작성자: Sage (LLM Specialist)
> 목적: 시스템 프롬프트 변경 시 회귀 감지 기준으로 사용

---

## 사용 방법

1. 프롬프트를 변경할 때마다 동일한 입력으로 실제 앱을 호출해 아래 "기대 출력"과 비교한다.
2. 감정 분류(`emotion`)는 정확히 일치해야 한다. 다른 분류가 나오면 회귀로 판단한다.
3. 공감 응답(`empathy`)은 동일 문장이 아닌 어조/방향성으로 평가한다:
   - 조언/해결책을 제시하는가? — 포함 시 실패
   - 따뜻한 대화체인가? — 임상적/딱딱하면 실패
   - 150자 이하인가? — 초과 시 실패
   - 한국어 존댓말인가? — 반말이면 실패

## 샘플 1 — 기쁨

### 입력 (사용자 일기)

```
오늘 팀장님한테 칭찬을 받았어요. 열심히 준비한 발표가 잘 됐고, 사람들도 좋은 반응을 보여줬어요. 집에 오는 길에 괜히 발걸음이 가벼웠어요.
```

### 기대 감정 분류

```json
"emotion": "기쁨"
```

(또는 "뿌듯함" — 두 감정 모두 허용 범위. 이 입력은 성취감이 동반된 기쁨이므로 둘 다 유효)

### 기대 공감 응답 방향

- 함께 기뻐하는 어조
- 발표 준비의 노력을 인정하는 내용
- "잘 됐다", "정말 기뻐요" 계열의 따뜻한 반응
- 조언 없음, "다음엔 더~" 같은 미래 지향적 제언 없음

### 샘플 기대 응답 예시

```json
{
  "emotion": "기쁨",
  "empathy": "오늘 정말 뿌듯한 하루셨겠어요. 열심히 준비한 것이 빛을 발한 거니까 충분히 기뻐하셔도 돼요. 오늘 그 기분 오래오래 간직하세요."
}
```

## 샘플 2 — 슬픔

### 입력 (사용자 일기)

```
오랫동안 친하게 지내던 친구가 갑자기 연락을 끊었어요. 이유도 모르겠고, 물어봐도 답장이 없어요. 뭘 잘못한 건지 계속 생각하게 돼요.
```

### 기대 감정 분류

```json
"emotion": "슬픔"
```

(또는 "불안" — 이유를 모르는 상태에서 자책하는 패턴은 불안과 혼란도 가능. 슬픔 또는 불안이면 허용)

### 기대 공감 응답 방향

- 상실감과 당혹감을 인정하는 어조
- 이유를 찾으라거나 먼저 연락해보라는 조언 없음
- "그런 상황이 얼마나 힘든지" 공감 중심

### 샘플 기대 응답 예시

```json
{
  "emotion": "슬픔",
  "empathy": "이유도 모른 채 소중한 사람과 멀어지는 건 정말 마음이 아프죠. 자꾸 생각하게 되는 게 당연해요. 혼자 너무 많이 자책하지 않으셨으면 해요."
}
```

## 샘플 3 — 스트레스

### 입력 (사용자 일기)

```
마감이 내일인데 아직 절반도 못 했어요. 오늘 회의가 세 개나 있어서 제대로 집중을 못 했고, 저녁도 못 먹었어요. 지쳐서 아무것도 하기 싫은데 해야 해요.
```

### 기대 감정 분류

```json
"emotion": "스트레스"
```

(또는 "피로" — 피로와 스트레스가 혼재. 둘 다 허용)

### 기대 공감 응답 방향

- 과부하 상황에 대한 공감
- "그래도 해야죠", "시간 관리를 잘 했으면" 같은 조언 없음
- 지쳐있는 상태 자체를 인정하는 내용

### 샘플 기대 응답 예시

```json
{
  "emotion": "스트레스",
  "empathy": "오늘 하루 정말 많이 힘드셨겠어요. 밥도 못 드시고 그 많은 걸 감당하셨으니 몸도 마음도 지치셨을 것 같아요. 수고 정말 많으셨어요."
}
```

## 샘플 4 — 불안

### 입력 (사용자 일기)

```
다음 주에 면접이 있는데 자꾸 이상한 상상을 해요. 떨어질 것 같고, 준비를 아무리 해도 부족한 것 같고, 어젯밤에도 잠을 잘 못 잤어요.
```

### 기대 감정 분류

```json
"emotion": "불안"
```

### 기대 공감 응답 방향

- 면접 전 불안감을 자연스러운 것으로 인정
- "더 준비하면 돼요", "잘 될 거예요" 같은 격려/조언 없음
- 불안한 마음 자체를 옆에서 함께 느끼는 어조

### 샘플 기대 응답 예시

```json
{
  "emotion": "불안",
  "empathy": "그 불안한 마음이 얼마나 무거울지 느껴져요. 잠도 제대로 못 주무시고 힘드셨겠어요. 오늘만큼은 그 걱정을 잠깐 내려놓고 쉬셔도 괜찮아요."
}
```

## 샘플 5 — 혼란

### 입력 (사용자 일기)

```
회사를 그만둬야 할지 모르겠어요. 지금 일이 너무 힘들긴 한데 막상 그만두면 어떻게 될지 무서워요. 계속 있어도 문제고 나가도 문제 같고 머릿속이 복잡해요.
```

### 기대 감정 분류

```json
"emotion": "혼란"
```

(또는 "불안" — 결정 장애와 미래 불확실성이 혼재. 둘 다 허용)

### 기대 공감 응답 방향

- 이러지도 저러지도 못하는 상황 자체에 공감
- "어떤 선택이 맞는지 생각해봐요" 같은 결정 유도 없음
- 복잡한 감정을 가져도 된다는 허용하는 어조

### 샘플 기대 응답 예시

```json
{
  "emotion": "혼란",
  "empathy": "어느 쪽을 선택해도 불안한 상황이라 정말 머릿속이 복잡하실 것 같아요. 그 갈등 자체가 그만큼 진지하게 생각하고 계신다는 거예요. 지금 당장 답을 내지 않아도 괜찮아요."
}
```

## 회귀 판단 기준 요약

| 검사 항목 | 합격 기준 | 실패 예시 |
|----------|----------|---------|
| 감정 분류 정확도 | 기대 감정 또는 허용 대안과 일치 | 스트레스 입력에 "기쁨" 반환 |
| 조언 부재 | empathy에 해결책/조언 없음 | "더 열심히 준비하면 돼요" |
| 길이 | 50자 이상 150자 이하 | 단일 10자짜리 문장 |
| 언어/존댓말 | 한국어, ~요 어미 | 영어 응답, "힘들었겠다" 반말 |
| JSON 형식 | `{"emotion": "...", "empathy": "..."}` 정확히 | 마크다운 코드 블록으로 감싼 JSON |
| 감정 목록 준수 | 13개 카테고리 중 하나 | "그리움", "외로움" 동의어 사용 |
