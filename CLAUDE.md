# Claude Code 지침

## Notion 일지 자동 기록

대화 중 아래 상황에서 **자동으로** `/notion-daily --log "내용"` 을 호출한다. 사용자에게 먼저 물어보지 않고 바로 실행한다.

### 자동 기록 트리거

| 상황 | 예시 |
|------|------|
| 아키텍처/구조 결정 | 컴포넌트 분리 방식, 상태 관리 선택, API 설계 방향 |
| 버그 원인 파악 | 재현 조건, 근본 원인, 임시 해결책 |
| 기술적 트레이드오프 발견 | A vs B 비교, 성능/유지보수 충돌 |
| 중요한 제약 발견 | 라이브러리 한계, 브라우저 호환성, 빌드 제약 |
| 작업 맥락 전환 | 현재 작업을 마치고 다른 작업으로 넘어갈 때 |

### 기록 형식 (--log 내용)

한 줄 요약: `[주제] 핵심 내용`

예시:
- `[아키텍처] Badge 컴포넌트는 variant prop 대신 개별 컴포넌트로 분리하기로 결정 (Figma 구조와 일치)`
- `[버그] Tailwind purge가 동적 클래스를 제거함 — safelist 또는 정적 클래스 문자열로 해결`
- `[트레이드오프] useMemo 적용 시 메모이제이션 비용이 실제 계산보다 클 수 있음 (항목 < 100개 기준)`

### 작업 시작/전환 시

새로운 주요 작업을 시작하거나 맥락이 바뀔 때 `/notion-daily --update "내용"` 도 함께 호출한다.
